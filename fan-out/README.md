**Fan-out** — распределение задач/значений из одного входного канала между несколькими

```
                ┌─→ worker1
input channel ──┼─→ worker2
                └─→ worker3
```

Несколько горутин читают из одного и того же канала. Ключевое свойство: каждое значение достаётся ровно одному воркеру (не копируется всем, как в tee) — они конкурируют за значения.

**Зачем нужен**

Параллельная обработка CPU-bound или I/O-bound задач. Один читатель канала обрабатывает значения строго последовательно — если обработка одного элемента занимает время, весь поток тормозится. Fan-out распределяет эту нагрузку между несколькими горутинами, которые могут выполняться на разных ядрах CPU или параллельно ждать I/O (сеть, диск).

```go
in := generator(nums...)

// fan-out
outs := fanOut(in, 3, expensiveComputation)

// fan-in
merged := fanIn(outs...)

for result := range merged {
    fmt.Println(result)
}
```


fan = веер (разветвление)

out = наружу (от одного источника к многим)
